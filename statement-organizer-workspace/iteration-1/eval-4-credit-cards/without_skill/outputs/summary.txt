PROJECT OVERVIEW
================================================================================
This project implements a comprehensive solution for processing and organizing
credit card statements from three different financial institutions:
  1. American Express (Business Card)
  2. Chase (Personal Card)
  3. Visa

The solution works WITHOUT a pre-built skill, implementing format detection
and organization logic from scratch.

APPROACH WITHOUT SKILL
================================================================================

The solution demonstrates how to handle three different credit card formats
without using a pre-built skill by:

1. DETECTION THROUGH PATTERN MATCHING
   - Identify each card type by distinctive headers
   - Recognize specific field arrangements
   - Calculate confidence scores based on feature matches
   - Handle ambiguous cases with threshold logic

2. FORMAT-SPECIFIC MAPPINGS
   Each card type has unique characteristics:

   AMERICAN EXPRESS BUSINESS
   - Header: "AMERICAN EXPRESS" or "AMEX"
   - Card prefix: 37
   - Fields: Date | Description | Category | Amount
   - Key feature: Business expense categories, rewards points
   - Date format: MM/DD/YYYY or MMM DD, YYYY
   - Delimiter: Comma (CSV)
   - Line endings: CRLF

   CHASE PERSONAL
   - Header: "CHASE" or "JPMORGAN CHASE"
   - Card prefix: 4
   - Fields: Posted Date | Description | Location | Amount | Type
   - Key feature: Merchant locations, Ultimate Rewards
   - Date format: MM/DD/YYYY (standardized)
   - Delimiter: Comma (CSV)
   - Line endings: LF

   VISA
   - Header: "VISA" or "VISA INC"
   - Card prefix: 4
   - Fields: Date | Description | Reference | Dr/Cr | Amount
   - Key feature: Debit/Credit indicators, running balance
   - Date format: DD/MM/YYYY or MM/DD/YYYY (flexible)
   - Delimiter: Comma or pipe
   - Line endings: CRLF

3. PROCESSING PIPELINE
   Input → Detection → Classification → Organization → Output

   Step 1: Collect files from two source directories
   Step 2: Detect card type using pattern matching
   Step 3: Assign to appropriate output folder
   Step 4: Generate processing reports
   Step 5: Store metadata and results

4. CONFIDENCE SCORING
   Detection uses weighted scoring:
   - Header match: +40 points
   - Feature match: +10 points per feature found
   - Format alignment: +5 points per field match
   - Confidence = (total_score / max_possible) × 100%

5. ORGANIZATION STRUCTURE
   /Statements/Organized/
   ├── AmEx_Business/        (American Express statements)
   ├── Chase_Personal/       (Chase statements)
   ├── Visa/                 (Visa statements)
   └── Unknown/              (Unclassified or ambiguous)

6. ERROR HANDLING
   - Unreadable files logged and moved to Unknown folder
   - Missing features flagged in confidence score
   - Low-confidence files isolated for manual review
   - Full audit trail maintained

EXTENSIBILITY
================================================================================

The framework is designed to support additional card types:

1. Add new format mapping JSON file
2. Register in CardFormatDetector.load_mappings()
3. Define distinctive features
4. Update detection rules
5. Create output folder in StatementOrganizer

Example for adding a new card type:
  - Create: amex_consumer_format_mapping.json
  - Add to _load_mappings() method
  - Define unique patterns in identifier_patterns
  - StatementOrganizer automatically handles new type
