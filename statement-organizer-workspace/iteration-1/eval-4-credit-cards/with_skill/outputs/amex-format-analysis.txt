DETECTED COLUMN STRUCTURE

7. Amount (USD) (Transaction Amount) **AMOUNT COLUMN**

COLUMN ANALYSIS
===============

Card Member:
  - Type: String (Name)
  - Sample: "Company Acme Inc"
  - Purpose: Identifies cardholder/account holder
  - Mapping: cardholder_column
  - Business-specific: YES

Card Number:
  - Type: String (Partial PII)
  - Sample: "374245XXXXXX1234"
  - Purpose: Card identifier (masked for security)
  - Mapping: card_number_column
  - Frequency: One per statement, consistent

Reference:
  - Type: String (Alpha-numeric)
  - Sample: "TXN20260310"
  - Purpose: Unique transaction identifier
  - Mapping: reference_column
  - Pattern: TXN[YYYYMMDD]

Posting Date:
  - Type: Date (YYYY-MM-DD)
  - Sample: "2026-03-10"
  - Purpose: Transaction settlement date
  - Mapping: date_column (PRIMARY)
  - Essential: YES
  - Date Format: ISO 8601

Description:
  - Type: String (Merchant Name)
  - Sample: "DELTA AIRLINES - FLIGHT 1234", "MARRIOTT HOTELS - NEW YORK"
  - Purpose: Transaction description/merchant
  - Mapping: merchant_column (PRIMARY)
  - Pattern: MERCHANT_NAME - ADDITIONAL_DETAIL
  - Business-category indicators: Airlines, Hotels, Car Rentals, Software, Office supplies

Card Present:
  - Type: Boolean (Yes/No)
  - Sample: "Yes", "No"
  - Purpose: Indicates if card was physically present
  - Mapping: card_present_column
  - Business indicator: Some transactions are online (SaaS), others in-person (travel)

Amount (USD):
  - Type: Currency (Decimal)
  - Sample: "1245.00", "3850.50", "247.89"
  - Purpose: Transaction amount in USD
  - Mapping: amount_column (PRIMARY)
  - Format: ####.## (no currency symbol in CSV)
  - Note: All amounts are positive (debits)
