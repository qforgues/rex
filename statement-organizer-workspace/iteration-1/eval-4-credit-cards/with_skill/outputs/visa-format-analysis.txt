COLUMN ANALYSIS
===============

Date:

  - Sample: "2026-03-02", "2026-03-23"

  - Mapping: date_column (PRIMARY)
  - Date Format: ISO 8601 (standardized)
  - Essential: YES
  - Frequency: One per transaction

Transaction:
  - Type: String (Transaction Type)
  - Sample: "DEBIT" (all in this statement)
  - Purpose: Transaction classification
  - Mapping: transaction_type_column
  - Typical values: DEBIT (charge), CREDIT (refund)
  - Frequency: All sample transactions are DEBIT (charges)
  - Business value: Allows filtering for different transaction types

Amount:
  - Type: Currency (Negative values)
  - Sample: "-125.50", "-45.99", "-320.00", "-188.00"
  - Purpose: Transaction amount in USD
  - Mapping: amount_column (PRIMARY)
  - Format: Negative values for debits/charges
  - Sign convention: Negative = spending, Positive would be credits/refunds
  - Currency: USD (implicit, Visa US card)
  - Range: -$34.99 to -$320.00

Merchant Code:
  - Type: Integer (MCC - Merchant Category Code)
  - Sample: 5411, 7832, 6211, 5814, 4722, 3000, 7298, 5733, 5812
  - Purpose: Industry classification (standardized by card networks)
  - Mapping: merchant_code_column
  - Format: 4-digit numeric code

      * 7832: Pharmacies (CVS)
      * 6211: Securities Brokers (Stock trading)
      * 5814: Fast Food/Restaurants (Sweetwater)
      * 4722: Travel/Ticket Agencies (TicketMaster)
      * 3000: Airlines (Airline ticket booking)
      * 7298: Book Stores (Amazon/Bookstore)
      * 5733: Computer Peripheral Shops
      * 5812: Eating Places/Restaurants (Steakhouse)
  - Business value: Standardized categorization independent of merchant name

Merchant Name:
  - Type: String (Merchant Name)
  - Sample: "KROGER SUPERMARKET", "CVS PHARMACY", "TICKETMASTER EVENTS", "STEAKHOUSE DINING"
  - Purpose: Transaction description/merchant identifier
  - Mapping: merchant_column (PRIMARY)
  - Pattern: MERCHANT_NAME + CATEGORY_DESCRIPTOR
  - Format: ALL CAPS with descriptive suffix
  - Consumer-category indicators: Grocery stores, Pharmacies, Airlines, Entertainment, Dining, Retail

City:
  - Type: String (City Name)
  - Sample: "Atlanta", "New York", "Chicago", "Seattle", "Tampa", "Miami", "Dallas"
  - Purpose: Geographic location of transaction
  - Mapping: city_column
  - Geographic distribution: Multiple US cities

Country:
  - Type: String (Country Name)
  - Sample: "United States" (all in this statement)
  - Purpose: Geographic country of transaction
  - Mapping: country_column
  - Value: All transactions domestic (US)
  - Business value: International vs domestic classification
  - Fraud indicator: Unexpected countries in transaction list
