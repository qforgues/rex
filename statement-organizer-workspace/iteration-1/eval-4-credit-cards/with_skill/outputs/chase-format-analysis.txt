COLUMN ANALYSIS
===============

Transaction Date:
  - Type: Date (MM/DD/YYYY)
  - Sample: "03/01/2026", "03/22/2026"
  - Purpose: Date transaction was initiated
  - Mapping: date_column (PRIMARY)
  - Date Format: MM/DD/YYYY (US format)
  - Essential: YES
  - Note: Distinct from Post Date

Post Date:
  - Type: Date (MM/DD/YYYY)
  - Sample: "03/02/2026", "03/23/2026"
  - Purpose: Date transaction settled/posted to account
  - Mapping: post_date_column
  - Typically 1 day after Transaction Date
  - Optional: Used for reconciliation

Description:
  - Type: String (Merchant Name with ID)
  - Sample: "TRADER JOE'S #234", "STARBUCKS COFFEE #5621", "NETFLIX SUBSCRIPTION"
  - Purpose: Merchant name and identifier
  - Mapping: merchant_column (PRIMARY)
  - Pattern: MERCHANT_NAME [#STORE_ID]
  - Consumer-category indicators: Groceries, Coffee, Pharmacy, Entertainment, Dining
  - Special cases: Subscriptions (NETFLIX), Online services (UBER)

Category:
  - Type: String (Predefined Category)
  - Sample: "Groceries", "Coffee Shops", "Pharmacy", "Entertainment", "Travel", "Dining"
  - Purpose: Transaction classification for budgeting
  - Mapping: category_column
  - Provided by: Chase (not merchant-derived)
  - Business value: Enables automatic expense categorization
  - Consumer-specific: YES - personal spending patterns

Type:
  - Type: String (Transaction Type)
  - Sample: "Purchase" (all in this statement)
  - Purpose: Transaction classification (Purchase, Refund, Fee, etc.)
  - Mapping: transaction_type_column
  - Frequency: Most transactions are "Purchase"
  - Optional: Allows filtering for different transaction types

Amount:
  - Type: Currency (Negative values)
  - Sample: "-87.43", "-5.45", "-32.18", "-15.99"
  - Purpose: Transaction amount in USD
  - Mapping: amount_column (PRIMARY)
  - Format: Negative values for charges (debits)
  - Sign convention: Negative = spending, Positive would be credits/refunds
  - Currency: USD (implicit, Chase US card)
