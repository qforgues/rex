================================================================================
HOW THE ALGORITHM WORKS
================================================================================

The algorithm uses a multi-signal SCORING approach:

SCORING COMPONENTS (0-150 points):
  1. Amount Matching (0-40 points)
     - Exact match (<=0.01): 40 points
     - Close match (<$1): 30 points
  
  2. Invoice ID Extraction (0-50 points)
     - ID found & matched: 50 points
     - No ID: 0 points
     Patterns recognized: INV-XXX, Invoice XXX, REF-XXX, bare numbers
  
  3. Client Name Matching (0-20 points)
     - Fuzzy match scoring (85% threshold)
     - Exact = 20 points, partial = proportional
  
  4. Date Proximity (0-10 points)
     - Payment within 30 days of invoice
     - Closer = more points

CLASSIFICATION THRESHOLDS:
  >= 95 points: EXACT MATCH (auto-safe to mark paid)
  70-94 points: PROBABLE MATCH (likely safe)
  45-69 points: AMBIGUOUS (requires manual review)
  < 45 points: NO MATCH (no corresponding invoice)


================================================================================
WHAT NEEDS MANUAL REVIEW
================================================================================

When confidence is AMBIGUOUS (45-69 points):

Follow 7-Step Verification Process:
  1. Check amount (exact or within tolerance?)
  2. Check client name (does it match?)
  3. Check timing (3-30 days after invoice?)
  4. Check reference (can you extract invoice number?)
  5. Apply decision matrix (do signals align?)
  6. Document decision (MATCH / NO MATCH / ESCALATE)
  7. Update FreshBooks and tracking

Decision Matrix Provided:
  All signals strong? → MATCH (safe to mark paid)
  Signals conflict? → ESCALATE (needs accountant review)
  Only amount matches? → LIKELY MATCH (verify with client)

See AMBIGUITY_HANDLING_GUIDE.txt for complete workflow.
