SECTION 1: HOW THE MATCHING ALGORITHM WORKS
================================================================================

The matching system uses a MULTI-STRATEGY SCORING APPROACH to connect Dart Bank
transactions to FreshBooks invoices. Instead of requiring a perfect match on a
single field, the algorithm evaluates multiple signals and assigns a score
(0-150 points) to each transaction-invoice pair.

SCORING COMPONENTS (Total: ~150 points possible):
===============================================

1. AMOUNT MATCHING (0-40 points)
   ├─ Exact match (difference <= $0.01):        40 points (HIGH CONFIDENCE)
   ├─ Close match (difference < $1):            30 points (MEDIUM CONFIDENCE)
   └─ No match (difference >= $1):              0 points

   Rationale: Most payments match exactly to invoice amounts. Accounting
   systems rarely have small rounding discrepancies. Exact amount matching
   is the strongest signal.

2. INVOICE ID EXTRACTION (0-50 points)
   ├─ Exact invoice ID found & matched:         50 points
   └─ No invoice ID found:                      0 points

   Pattern Recognition:
   - Looks for: "INV-2026-001" format
   - Looks for: "Invoice 2026-001" (extracts number, formats as INV-2026-001)
   - Looks for: "REF-2026-001" (extracts and validates)
   - Looks for: Standalone numbers (2026-001) in reference field

   Rationale: Bank reference fields often contain the invoice ID. If we can
   confidently extract and match an invoice ID, this is strong evidence.

3. CLIENT NAME/MERCHANT MATCHING (0-20 points)
   ├─ Exact name match:                         20 points (fuzzy_score = 1.0)
   ├─ Partial match (>85% similarity):          15-19 points
   ├─ Weak match (50-85% similarity):           5-14 points
   └─ No match:                                 0 points

   Rationale: Portal42 merchant names extract the client name (e.g.,
   "Portal42-Acme Corp" → "Acme Corp"). We compare this with FreshBooks
   client names using string similarity. Exact matches increase confidence.

4. DATE PROXIMITY (0-10 points)
   ├─ Payment within 3 days of invoice date:    10 points
   ├─ Payment within 30 days of invoice date:   5-9 points (decreasing)
   └─ Payment outside 30-day window:            0 points

   Rationale: Payments typically arrive within 3-30 days of invoice. We
   penalize transactions that occur before the invoice date or much later.
   This prevents matching old invoices to recent payments.


CLASSIFICATION THRESHOLDS:
==========================

Score >= 95:        EXACT MATCH
                    → Mark invoice as PAID immediately
                    → Confidence: 95%+
                    → Typical score: 115-120 points

Score 70-94:        PROBABLE MATCH
                    → Usually safe to mark as paid
                    → But verify manually for large amounts
                    → Confidence: 70-94%

Score 45-69:        AMBIGUOUS
                    → REQUIRES MANUAL REVIEW
                    → Multiple invoices have similar scores
                    → May be correct but not certain
                    → Confidence: 45-69%

Score < 45:         NO MATCH
                    → Transaction doesn't correspond to any invoice
                    → May be a fee, transfer, or invoice not yet in FreshBooks
                    → Confidence: <45%

SECTION 2: HANDLING AMBIGUITIES & EDGE CASES
================================================================================

AMBIGUITY SITUATION 1: Multiple Close Matches
----------------------------------------------
Problem: A transaction of $2,500 could match Invoice A ($2,500, score 115)
         AND Invoice B ($2,500, score 110)

Solution:
1. Calculate scores for ALL unmatched invoices
2. Rank by score (highest first)
3. Check if top score is SIGNIFICANTLY higher (>20 point gap)
   - If yes: Consider it probable/exact match
   - If no: Mark as AMBIGUOUS, flag both candidates for review
4. Lock matched invoices: Once matched, invoice cannot be matched again
   (prevents double-counting)

Example from data:
  DART-20260320-001 ($4,500)
  - Top candidate: INV-2026-008 (score 57.3) - AMBIGUOUS
  - 2nd candidate: INV-2026-008 (score 57.3) - CONFLICT!
  Action: FLAG FOR REVIEW - Check invoice dates and reference text


AMBIGUITY SITUATION 2: Partial Invoice ID Extraction
-----------------------------------------------------
Problem: Reference says "INV-003" but we have "INV-2026-003" in FreshBooks

Solution:
1. Extract "003" from reference
2. Try substring matching: "003" appears in "INV-2026-003"
3. If confident match found, proceed with other scoring
4. If unsure, lower confidence level (treat as middle name match, not ID match)

Example from data:
  DART-20260315-001 reference: "INV-003"
  Extracted: "INV-003"
  Matches: "INV-2026-003" (substring match of "003")
  Result: AMBIGUOUS (ID match partial, not confident)


AMBIGUITY SITUATION 3: Non-Portal42 Transactions
-------------------------------------------------
Problem: Bank statement includes transfers, fees, withdrawals (not client payments)

Solution:
1. Check merchant name for "Portal42" prefix
2. If NOT found: Classify as NO_MATCH
3. Do not attempt to match against invoices
4. Flag reason: "Non-Portal42 transaction (transfer/fee)"

Example from data:
  DART-20260327-001: "Transfer-Internal" $5,000
  Action: SKIP - Not a client payment


AMBIGUITY SITUATION 4: Missing or Incorrect Reference
------------------------------------------------------
Problem: Some transactions have no reference or vague reference ("2026-006")

Solution:
1. Still perform amount match (most reliable signal)
2. Use merchant name for fuzzy matching
3. Use date proximity as tie-breaker
4. Score may be lower (no ID match bonus), but still valid

Example from data:
  DART-20260312-001 reference: "2026-006"
  - Amount match: $1,500.75 (40 points)
  - Name match: "Creative Studio" (20 points)
  - Date: 4 days after invoice (8 points)
  - ID extraction: Incomplete (0 points)
  Total: 68.7 points → AMBIGUOUS
