================================================================================
HOW THE ALGORITHM WORKS (SIMPLE VERSION)
================================================================================

For each bank transaction:

1. EXTRACT KEY FIELDS
   - Amount: $2,500.00
   - Merchant: "Portal42-Acme Corp"
   - Reference: "INV-2026-001"
   - Date: 2026-03-01

2. COMPARE AGAINST ALL UNPAID INVOICES
   For each invoice, calculate match score:

   a) AMOUNT MATCH (0-40 points)
      If difference <= $0.01: 40 points (exact)
      If difference < $1: 30 points (close)
      Otherwise: 0 points

   b) INVOICE ID EXTRACTION (0-50 points)
      Extract invoice number from reference field
      If matches invoice ID: 50 points
      If not found: 0 points

   c) CLIENT NAME MATCH (0-20 points)
      Extract "Acme Corp" from "Portal42-Acme Corp"
      Compare to invoice client name using fuzzy matching
      Score based on similarity (0-20 points)

   d) DATE PROXIMITY (0-10 points)
      If transaction <= 30 days after invoice: some points
      Closer = more points, up to 10 for same day

3. CLASSIFY RESULT
   - Score >= 95: EXACT MATCH (mark as paid)
   - Score 70-94: PROBABLE MATCH (likely safe)
   - Score 45-69: AMBIGUOUS (needs review)
   - Score < 45: NO MATCH (check if invoice exists)

4. PREVENT DOUBLE-MATCHING
   Once invoice is matched to a transaction, "lock" it
   No other transactions can match the same invoice


================================================================================
HANDLING AMBIGUITIES
================================================================================

When a match is AMBIGUOUS (45-69 points), manually verify:

STEP 1: Check Amount
  Does bank transaction amount match invoice? If yes, strong signal.

STEP 2: Check Client Name
  Does merchant match invoice client? If yes, strong signal.

STEP 3: Check Date
  Is payment within 3-30 days of invoice? If yes (5-15 days best), likely match.

STEP 4: Check Reference
  Can you extract invoice number from bank reference?
  If yes (even partially), likely match.

STEP 5: Apply Decision Matrix
  If 3+ signals align: MATCH
  If signals conflict: ESCALATE to accountant

Example: $890.25 payment matches $890.25 invoice, client name matches,
         payment is 6 days after invoice, but reference uses "REF-" prefix
         instead of "INV-"
  → Likely MATCH (3 strong signals), but note unusual reference format

SUMMARY IN 3 SENTENCES
================================================================================

This evaluation implements a Python-based matching algorithm that connects
Dart Bank transactions to FreshBooks invoices using multi-signal scoring
(amount, invoice ID, client name, date proximity) and classifies each match
as exact (95%+ confidence), probable (70-94%), ambiguous (45-69%, needs review),
or no-match (<45%).

The sample data shows that standardizing the bank statement reference field
format (from inconsistent "INV-" / "Invoice" / "REF-" / bare-number styles to
always "INV-YYYY-NNN") would improve automatic matching confidence from 30.8%
to ~98%.

Complete documentation includes the algorithm explanation, 7-step manual review
process for ambiguous matches, three output reports, and four recommendations
for improving matching quality in order of impact and effort.
