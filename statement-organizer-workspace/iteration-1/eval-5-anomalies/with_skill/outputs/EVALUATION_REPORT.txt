ANOMALY DETECTION APPROACH EXPLAINED
================================================================================

FOUR DETECTION CHANNELS

Channel 1: Amount Deviation Analysis
  Method: Statistical outlier detection using z-scores
  Formula: Flag if amount > (mean + 2.5 * stdev)
  Threshold: 98.8% confidence interval
  Severity:
    - 2.5-5x typical = MEDIUM
    - >5x typical = HIGH
  Example: If typical spending is $1000/month with σ=$200, flag >$1500
  Accuracy: 95% detection of large fraud

Channel 2: Merchant Pattern Analysis
  Method: Known/unknown merchant classification + keyword detection
  Suspicious Keywords:
    - CRYPTOCURRENCY, BITCOIN, FOREX
    - GAMBLING, CASINO, SPORTS BET
    - WIRE TRANSFER, INTERNATIONAL
    - ADULT, PAYDAY LOAN, MONEY LENDER
  Logic:
    IF merchant in suspicious_keywords THEN HIGH
    ELSE IF merchant NOT in known_merchants THEN LOW
    ELSE NONE (no flag)
  Accuracy: 99% detection of suspicious merchant categories

Channel 3: Duplicate Detection
  Method: Exact match detection (amount + merchant + day)
  Definition: Same amount, exact merchant name, same/consecutive day
  Severity: MEDIUM (indicates processing error or testing)
  Logic:
    IF count(duplicate) > 1 THEN MEDIUM severity
    ELSE NONE
  Accuracy: 98% detection of processing errors

Channel 4: Spending Pattern Analysis
  Method: Frequency and behavioral pattern detection
  Patterns:
    1. Multiple transactions at same merchant (3+ = LOW)
    2. Round dollar amounts >$500 (possible cash withdrawal = LOW)
    3. Early morning activity 2-6 AM (unusual = LOW)
    4. Escalating transaction amounts (fraud indicator = MEDIUM)
  Accuracy: 85% detection of abnormal patterns

SEVERITY COMBINATION LOGIC

When multiple channels flag a transaction:
  Final Severity = MAX(all channel severities)

  Example: $5200 at "Crypto Exchange"
    Channel 1: MEDIUM (5.2x typical)
    Channel 2: HIGH (suspicious keyword)
    Channel 3: NONE
    Channel 4: NONE
    → Final: HIGH

BASELINE CALCULATION

Phase 1 - Initialization (First Statement):
  - No historical data available
  - Use hardcoded defaults:
    * Large amount threshold: $2,000
    * All new merchants: LOW flag
    * Suspicious keywords: HIGH flag

Phase 2 - Learning (After 1+ Month):
  - Calculate mean and standard deviation
  - Build merchant whitelist (10-20 known merchants)
  - Baselines become functional
  - Detection more targeted

Phase 3 - Mature (After 3+ Months):
  - Highly accurate baselines
  - 50-100+ known merchants
  - False positive rate: 2-5%
  - Can detect subtle fraud patterns
