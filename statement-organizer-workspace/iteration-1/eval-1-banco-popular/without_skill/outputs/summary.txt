3. PARSING LOGIC: HOW TRANSACTIONS ARE EXTRACTED
================================================================================

A. NUMERIC PARSING
   - Depositos and Retiros columns can contain:
     * Numeric values: "1500.50" → 1500.50
     * Zero indicators: "-" or "" → 0.0
   - Parser handles both representations seamlessly

B. TRANSACTION CLASSIFICATION
   Each parsed transaction is categorized into:

   - CATEGORY (Income/Expense/Fee/Other)
     Based on keywords in description field

   - TRANSACTION TYPE (specific type)
     * Direct Deposit (from "Deposito Directa")
     * Check Deposit (from "Deposito Cheque")
     * Transfer In (from "Transferencia Recibida")
     * Transfer Out (from "Transferencia Enviada" or "Transferencia a")
     * Debit Card Purchase (from "Compra Tarjeta Debito")
     * ATM Withdrawal (from "Retiro Cajero")
     * Bill Payment (from "Pago Factura")
     * Interest (from "Interes")
     * Fee (from "Comisión" or "Cargo")
     * Balance (opening/closing entries)

C. DATA VALIDATION
   - Date validation: Only rows with valid YYYY-MM-DD dates are parsed
   - Balance verification: Final balance matches opening + deposits - withdrawals
   - Numeric consistency: All amounts converted to float with 0.0 default
   - No header rows in data: Skip rows that match header names

D. ERROR HANDLING
   - Failed rows are skipped with warnings (non-blocking)
   - Partial parsing completes even if some rows fail
   - Comprehensive error messages logged for review
