SECTION 1: HOW TO LEARN A NEW BANK'S STATEMENT FORMAT (WITHOUT SKILL)
================================================================================

The approach consists of 5 key phases:

1. DISCOVERY PHASE
   - Load the CSV statement file
   - Extract column headers
   - Sample first N rows of data
   - Goal: Understand structure without assumptions

2. ANALYSIS PHASE
   - Header-based clue detection: Look for keywords like "date", "amount",
     "balance", "description", "merchant", etc.
   - Content-based pattern matching:
     * Check if column values match date patterns (YYYY-MM-DD, MM/DD/YYYY, etc.)
     * Validate numeric patterns for amounts (with/without decimals, separators)
     * Identify reference IDs using alphanumeric patterns
     * Categorize text fields as descriptions/merchants
   - Calculate confidence scores (0.0 to 1.0) for each detection
   - Goal: Map each column to a semantic field type

3. CLASSIFICATION PHASE
   - Assign field types to each column:
     * DATE: Transaction date
     * DESCRIPTION: Merchant/payee name
     * AMOUNT: Generic amount
     * DEBIT: Money out
     * CREDIT: Money in
     * BALANCE: Account balance
     * REFERENCE: Transaction ID/reference
   - Rank by confidence to identify must-have fields
   - Goal: Create a semantic understanding of the format

4. VALIDATION PHASE
   - Define rules for each field type:
     * Date formats (regex patterns for validation)
     * Numeric patterns and ranges
     * Required vs. optional fields
   - Test with sample data to ensure parsing works
   - Goal: Ensure detected format is usable

5. INTEGRATION PHASE
   - Create a mapping dictionary with:
     * Field mappings (column name -> field type)
     * Parsing rules (date format, decimal separators, currency)
     * Validation rules for each field
   - Store mapping in a registry for future use
   - Parse full statement using detected format
   - Normalize to standard transaction format
   - Goal: Enable automatic processing of future statements

================================================================================
SECTION 2: FORMAT DETECTION LOGIC
================================================================================

KEY ALGORITHMS:

A. Header-Based Detection
   For each column header:
   - Tokenize and lowercase the header
   - Check against keyword lists for each field type
   - Return field type + confidence if match found
   - Example: "Transaction Date" -> DATE (confidence: 0.9)

B. Content-Based Pattern Matching
   For each column's sample values:
   - Count matches against regex patterns for field type
   - Calculate match ratio (matches / total samples)
   - Threshold: >80% matches = classified as that type
   - Fallback to lower confidence if below threshold

C. Confidence Scoring
   - Header-based detection: High confidence (0.8-0.9)
   - Pattern-based detection: Medium confidence (0.7-0.85)
   - No match: Low confidence (0.5-0.6)
   - Final score: Average of all field confidences

D. Type Inference
   When both debit and credit columns present:
   - Check for "Type" or "Category" column
   - Use text patterns: "Credit", "Debit", "In", "Out"
   - Example: Amount + "Credit" type -> CREDIT field

================================================================================
SECTION 3: IMPLEMENTATION - FORMAT DETECTION CODE
================================================================================

File: format_detection.py
Main class: FormatDetector

Key methods:
1. load_csv(filepath)
   - Reads CSV file
   - Returns headers and data rows

2. detect_field_type(column_name, column_values, index)
   - Analyzes single column
   - Returns (FieldType, confidence)
   - Uses keyword + pattern matching

3. analyze_headers(headers, data_rows)
   - Processes all columns
   - Returns list of ColumnInfo objects
   - Each object has field type, confidence, sample values

4. generate_format_mapping()
   - Creates mapping dictionary from detected columns
   - Includes validation rules for each field
   - Sets required fields

5. parse_transactions(data_rows)
   - Uses detected format to parse CSV
   - Returns list of normalized transaction dicts
   - Converts field types appropriately

SECTION 6: INTEGRATION WITH EXISTING SYSTEM
================================================================================

To add NewBank to the statement organizer:

1. RUN DETECTION:
   detector = FormatDetector()
   headers, data = detector.load_csv("NewBank-March-2026.csv")
   columns = detector.analyze_headers(headers, data)
   mapping = detector.generate_format_mapping()

2. REGISTER FORMAT:
   registry = BankFormatRegistry()
   registry.register_format("NewBank", mapping)

3. PARSE STATEMENTS:
   parser = StatementParser(registry)
   transactions = parser.parse_statement("NewBank-March-2026.csv", "NewBank")

4. NORMALIZE:
   normalizer = TransactionNormalizer()
   normalized = [normalizer.normalize(tx) for tx in transactions]

5. SAVE MAPPING:
   with open("newbank_mapping.json", "w") as f:
       json.dump(mapping, f, indent=2)

FUTURE STATEMENTS:
   For future NewBank statements, the mapping is cached, so parsing is instant:
   parser.parse_statement("NewBank-April-2026.csv", "NewBank")
   # Uses cached format - no re-detection needed
