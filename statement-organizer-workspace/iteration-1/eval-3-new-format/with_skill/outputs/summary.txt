    - "Credit Amount" contains "credit" ✓

Validation Rules:
  - Essential columns required: date_column AND amount_column
  - If both found: Confidence = HIGH (auto-save)

  - NewBank: All essential + optional columns found → HIGH confidence

FORMAT MAPPING STORAGE
======================

Location: /Statements/.statement-formats.json
Format: JSON with one entry per bank

NewBank Entry:
{
  "NewBank": {
    "date_column": "Post Date",
    "amount_column": "Debit Amount",
    "merchant_column": "Merchant Name",
    "description_column": "Description",
    "debit_amount_column": "Debit Amount",
    "credit_amount_column": "Credit Amount",
    "file_pattern": "*NewBank*",
    "detected_fields": [...list of all 7 column headers...],
    "auto_detection_confidence": "high",
    "detection_timestamp": "2026-03-23T11:47:00Z",
    "learning_notes": "Format auto-detected successfully. Bank uses separate Debit/Credit..."
  }
}

Key Fields Explained:

- date_column: Which CSV column contains transaction dates
- amount_column: Primary amount column (used if no debit/credit split)
- merchant_column: CSV column with merchant/payee names
- description_column: CSV column with transaction descriptions
- debit_amount_column: For split format - column with debit amounts
- credit_amount_column: For split format - column with credit amounts
- file_pattern: Glob pattern to match future statements (e.g., "*NewBank*")
- detected_fields: Complete list of all columns found (for reference)
- auto_detection_confidence: "high" = automatic, "low" = manual identification needed
- detection_timestamp: When this format was learned (for audit trail)

TECHNICAL IMPLEMENTATION NOTES
=============================

The skill's StatementProcessor class implements:

1. detect_bank(file_path)
   - Returns bank name from filename or CSV headers
   - Used to identify which format mapping to apply

2. learn_format(file_path, bank_name)
   - Reads CSV, calls _auto_detect_columns()
   - Falls back to _interactive_format_setup() if needed
   - Saves mapping via _save_formats()

3. _auto_detect_columns(fieldnames, bank_name)
   - Performs pattern matching on CSV headers
   - Returns mapping dict with all detected columns
   - Validates essential columns exist

4. parse_statement(file_path, bank_name)
   - Loads format from memory
   - Iterates rows, calling _extract_transaction()
   - Returns list of transaction dicts

5. _extract_transaction(row, format_info)
   - Uses format_info to extract fields from row
   - Handles debit/credit parsing
   - Cleans amounts (removes $, commas)
   - Returns normalized transaction object
